## Architecture Decisions

- **No circular imports:** `auth.ts` uses a bare `ky.create()` instance (not the `api` singleton from `api.ts`). `api.ts` imports from `auth.ts`. Dependency direction is one-way.
- **Refresh-then-logout pattern:** On 401, the API client tries a silent token refresh before forcing logout. This avoids unnecessary session interruptions when the access token (15min TTL) expires mid-session.
- **Zod password rules on register:** `min(8)` + uppercase + digit. Matches the backend's `Test1234!` test case from the phase plan. Login only requires `min(8)` (whatever the user set at registration time).
- **Refresh token in SecureStore:** Stored under `auth_refresh_token`. `loadToken()` restores it so refresh works after app restarts.
